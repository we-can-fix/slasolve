## ✅ 治理檢查清單

<!-- 請確認以下所有項目 -->

### 基本要求
- [ ] 程式碼遵循專案的編碼規範
- [ ] 所有文檔與註解使用繁體中文
- [ ] 提交訊息符合 Conventional Commits 格式
- [ ] 已閱讀並理解 CONTRIBUTING.md
- [ ] 變更不會導致破壞性更新（或已明確說明）

### 程式碼品質
- [ ] 程式碼已通過 linter 檢查
- [ ] 已添加或更新相關測試
- [ ] 所有測試都已通過
- [ ] 程式碼覆蓋率達到要求（≥80%）
- [ ] 沒有引入新的技術債務

### 安全性
- [ ] 已檢查並修復所有安全漏洞
- [ ] 沒有硬編碼的敏感資訊（密碼、密鑰、令牌）
- [ ] 容器映像使用 SHA256 digest
- [ ] 已設置適當的安全上下文（runAsNonRoot, readOnlyRootFilesystem）
- [ ] 已審查第三方依賴的安全性

### 策略合規
- [ ] Kubernetes 資源包含所有必要標籤
  - `namespace.io/managed-by`
  - `namespace.io/domain`
  - `namespace.io/environment`
  - `app`, `version`
- [ ] UAV/AD 系統包含特定標籤（如適用）
  - `uav.io/system`
  - `uav.io/safety-level`
- [ ] 資源設置了適當的限制（CPU, Memory）
- [ ] 已通過策略驗證（conftest/OPA）

### 文檔
- [ ] 已更新相關文檔
- [ ] 已更新 CHANGELOG.md（如果是新版本）
- [ ] API 變更已記錄在文檔中
- [ ] 添加了使用範例（如果是新功能）

### 測試
- [ ] 單元測試已添加或更新
- [ ] 整合測試已通過
- [ ] 策略測試已通過
- [ ] 已在本地環境驗證變更
